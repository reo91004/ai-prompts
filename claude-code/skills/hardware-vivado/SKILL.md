Apply hardware/Vivado gates. Check tool version, target part, constraints, simulation logs, synthesis logs, implementation logs, timing reports, utilization reports, bitstream evidence when applicable, testbench limitations, and whether functionality was preserved. Do not claim timing closure without implementation logs.

This skill governs RTL, synthesis, implementation, timing, and utilization evidence. When the task includes physical acquisition from instruments or boards, also use `hardware-capture-integrity`; do not treat a successful bitstream build as capture-integrity evidence.
