## Policy
Do not produce or accept placeholder work.

Read `references/placeholder_hardcoding_policy.md` for strict rules.

For code changes: implement real behavior, add validation and error handling, avoid silent fallback, avoid test-only hardcoding, run relevant checks when possible, and document limitations honestly.
