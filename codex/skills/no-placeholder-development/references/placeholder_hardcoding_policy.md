# Placeholder, Fallback, and Hardcoding Policy

Forbidden unless explicitly requested as a planning-only stub:
TODO, FIXME, placeholder, dummy, stub, temporary, hack, fake output, fake metric, fake test pass, fixed return value that ignores real input, fallback that hides real failure, disabled assertion used to pass tests, test-mode branch that bypasses logic, magic constants without documented source.

Allowed: named constants traceable to a spec, paper, config, invariant, or protocol; explicit NotImplementedError or safe blocking behavior when a feature is truly unsupported; documented limitations in carry-over notes.
